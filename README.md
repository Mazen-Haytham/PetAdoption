## вңЁ About the Project

**PetPal Adoption** is a full-stack web platform designed to bridge the gap between shelter animals and loving homes. Whether you're looking to adopt a dog, cat, or any other companion, PetPal makes the journey seamless вҖ” from browsing available pets to submitting an adoption request.

> рҹҗҫ *"Every pet deserves a forever home."*

## вҡҷпёҸ Getting Started

### Prerequisites

Make sure you have the following installed:

- **Node.js** `>= 20.x` вҖ” [Download](https://nodejs.org/)
- **npm** or **yarn**
- **.NET SDK** `9.0` вҖ” [Download](https://dotnet.microsoft.com/download)
- **Docker & Docker Compose** вҖ” [Download](https://www.docker.com/products/docker-desktop/) *(recommended for full-stack setup)*
- **Git** вҖ” [Download](https://git-scm.com/)

### Clone the Repository

```bash
git clone https://github.com/Mazen-Haytham/PetAdoption.git
cd PetAdoption
```

## рҹ–ҘпёҸ Frontend Setup

```bash
# Navigate to the frontend directory
cd Frontend/pet-pal-adoption

# Install dependencies
npm install

# Start the development server
npm start
```

The app will be running at `http://localhost:3000` рҹҺү

---

## рҹ”§ Backend Setup

```bash
# Navigate to the backend directory
cd backend

# Restore NuGet packages
dotnet restore

# Apply database migrations (if applicable)
dotnet ef database update

# Run the API server
dotnet run
```

The API will be available at `http://localhost:5000` or `https://localhost:5001` рҹ”Ң

> **Note:** Update the connection string in `appsettings.json` to point to your local database before running.

### Quick Start with Docker Compose

**1. Set up environment variables**

Create a `.env` file in the project root:

```env
MSSQL_SA_PASSWORD=YourStrongPassword123!
JWT_TOKEN=MySuperSecureAndRandomKeyThatLooksJustAwesomeAndNeedsToBeVeryVeryLong!!!111oneeleven!
```

**2. Launch the entire stack**

```bash
docker compose up --build
```

| Service | URL |
|---------|-----|
| рҹ–ҘпёҸ Frontend | `http://localhost:5173` |
| рҹ”Ң Backend API | `http://localhost:8080` |
| рҹ—„пёҸ SQL Server | `localhost:1433` |
| вҡЎ Redis | `localhost:6379` |

**3. Stop the stack**

```bash
docker compose down
```

To also remove volumes (wipes the database):

```bash
docker compose down -v
```

### рҹҗі Frontend Dockerfile

Multi-stage build: **Node 20** compiles the React app вҶ’ **Nginx Alpine** serves the static output.

```dockerfile
# в”Җв”Җ Stage 1: Build в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci
COPY . .
RUN npm run build

# в”Җв”Җ Stage 2: Runtime в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FROM nginx:alpine AS runtime
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### рҹҗі Backend Dockerfile

Multi-stage build: **.NET 9 SDK** compiles and publishes вҶ’ **.NET 9 ASPNet runtime** runs the app lean.

```dockerfile
# в”Җв”Җ Stage 1: Build в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FROM mcr.microsoft.com/dotnet/sdk:9.0 AS build
WORKDIR /src
COPY backend.csproj .
RUN dotnet restore
COPY . .
RUN dotnet publish -c Release -o /app/publish --no-restore

# в”Җв”Җ Stage 2: Runtime в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
FROM mcr.microsoft.com/dotnet/aspnet:9.0 AS runtime
WORKDIR /app
COPY --from=build /app/publish .
EXPOSE 8080
ENV ASPNETCORE_ENVIRONMENT=Docker
ENTRYPOINT ["dotnet", "backend.dll"]
```

### рҹҗі docker-compose.yml

Full orchestration with health checks, dependency ordering, shared volumes, and environment wiring:

```yaml
version: '3.8'

networks:
  petadoption-network:
    driver: bridge

services:
  mssql:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      ACCEPT_EULA: "Y"
      MSSQL_SA_PASSWORD: "${MSSQL_SA_PASSWORD:-StrongPassword123!}"
    ports:
      - "1433:1433"
    volumes:
      - mssql_data:/var/opt/mssql
    networks:
      - petadoption-network
    healthcheck:
      test: ["CMD-SHELL", "/opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P \"${MSSQL_SA_PASSWORD:-StrongPassword123!}\" -C -Q 'SELECT 1' || exit 1"]
      interval: 10s
      retries: 10
      start_period: 30s

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    networks:
      - petadoption-network
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      retries: 5

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8080:8080"
    networks:
      - petadoption-network
    environment:
      ASPNETCORE_ENVIRONMENT: Docker
      ASPNETCORE_URLS: http://+:8080
      ConnectionStrings__DefaultConnection: >
        Data Source=mssql,1433;
        Initial Catalog=PetAdoptionDb;
        User Id=SA;
        Password=${MSSQL_SA_PASSWORD};
        Encrypt=True;
        TrustServerCertificate=True;
      ConnectionStrings__Redis: "redis:6379"
      AppSettings__Token: "${JWT_TOKEN}"
      AppSettings__Issuer: "PetAdoption"
      AppSettings__Audience: "PetAdoptionAudience"
      AppSettings__TokenExpiryMinutes: "15"
      FrontendUrl: "http://frontend"
    depends_on:
      mssql:
        condition: service_healthy
      redis:
        condition: service_healthy
    volumes:
      - ./backend/wwwroot/uploads/pets:/app/wwwroot/uploads/pets

  frontend:
    build:
      context: ./Frontend/pet-pal-adoption
      dockerfile: Dockerfile
    ports:
      - "5173:80"
    networks:
      - petadoption-network
    depends_on:
      backend:
        condition: service_started
    volumes:
      - ./backend/wwwroot/uploads/pets:/usr/share/nginx/html/uploads/pets

volumes:
  mssql_data:
```

> рҹ’Ў **Shared Volume:** Both `frontend` and `backend` mount `./backend/wwwroot/uploads/pets` вҖ” this means pet images uploaded via the API are immediately served by Nginx without any file copying.
